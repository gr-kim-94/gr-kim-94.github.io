# 포트폴리오(garam.github.io) 구성 계획서

이 문서는 `al-folio` 테마를 기반으로 한 개인 포트폴리오 사이트를 효과적으로 구성하기 위한 가이드입니다.

## 1. 핵심 컨텐츠 전략

| 컨텐츠 종류           | 위치              | 주요 포함 내용                                         |
| :-------------------- | :---------------- | :----------------------------------------------------- |
| **자기소개 (About)**  | `_pages/about.md` | 본인의 정체성을 나타내는 한 문장, 주요 기술, 관심 분야 |
| **대표 프로젝트**     | `_projects/`      | 기술적 도전 과제, 문제 해결 사례, 데모 GIF/이미지      |
| **경력 및 학력 (CV)** | `_pages/cv.md`    | 역순으로 정리된 이력, 스킬 셋 시각화                   |
| **기술 블로그**       | `_posts/`         | 학습 기록, 문제 해결 과정, 튜토리얼                    |
| **뉴스/업데이트**     | `_news/`          | 외부 활동, 수상, 자격증 취득 등 짧은 소식              |

## 2. 세부 설정 가이드

### 2.1 사이트 기본 정보 (`_config.yml`)

- `title`: 사이트의 공식 명칭 (예: "가람의 기술 블로그")
- `first_name`, `last_name`: 본인의 이름 설정
- `description`: 검색 엔진에 노출될 사이트 설명
- `socials`: GitHub, LinkedIn, Twitter 등 본인의 소셜 미디어 연결

### 2.2 자기소개 페이지 (`_pages/about.md`)

- **프로필 사진**: `assets/img/prof_pic.jpg` 경로에 본인 사진을 배치하고 연결하세요.
- **핵심 키워드**: 상단에 본인을 정의하는 키워드(예: Backend Developer, Data Scientist)를 강조하세요.

### 2.3 프로젝트 섹션 (`_projects/`)

- 새로운 프로젝트를 추가할 때마다 `_projects/` 폴더에 `.md` 파일을 만듭니다.
- **YAML Frontmatter** 중요 필드:
  - `importance`: 숫자가 낮을수록 상단에 노출됩니다.
  - `category`: 'work', 'personal' 등으로 분류 가능합니다.
  - `img`: 프로젝트 썸네일 경로를 설정합니다.

### 2.4 블로그 (`_posts/`)

- 파일명 형식: `YYYY-MM-DD-title.md`를 반드시 준수하세요.
- 태그(`tags`)와 카테고리(`categories`)를 활용해 효율적으로 분류하세요.

## 3. 향후 작업 순서 (추천)

1. **Phase 1: 설정 최적화**
   - `_config.yml`의 기본 정보 수정
   - `_pages/about.md`의 예시 텍스트를 본인 내용으로 교체

2. **Phase 2: 프로젝트 기록**
   - 현재까지 진행한 프로젝트 중 가장 핵심적인 2개를 먼저 업로드

3. **Phase 3: 이력서(CV) 연동**
   - `_pages/cv.md`를 수정하거나 JSON Resume 포맷을 활용해 역량 정리

4. **Phase 4: 배포 및 검증**
   - 로컬에서 Docker를 활용해 실행 및 레이아웃 확인 (`docker compose up`)
   - GitHub Pages에 배포 후 모바일 환경 확인
